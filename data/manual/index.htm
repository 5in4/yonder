Yonder User Documentation

Concept

Yonder is an easy to use application to create realistic sound amtospheres for pen&paper roleplaying games.

It differentiates between four basic types of events: Music, Atmosphere, Singleshots and Sfx. These types are individually mixed together to create the atmosphere suiting your needs. The exact purpose of each of these events will be described later.

These four event types are always part of a project, which is located in a folder on your file system.

Each event contains objects, which are a pool of tracks that belong together (think of playlists in media players like iTunes or Winamp).

Yonder does not come with any sound files, but with a Resource Browser filled with links to online resources which makes your first steps at building your own sound library much easier.

Note: Yonder is still in development. It is limited in its functionality, but provides the basic features needed for ambience sound generation. Features may improve and or change in their behaviour in future releases.

Using Yonder

Start

When starting Yonder for the first time, it will ask you for a project folder. Pick an empty folder where you want your project to be located. Inside the project folder, three subfolders will be created, one for your music files, one for atmospheres and one for singleshots and sfx files.

Everything you need to run a project is located inside this project folder. If you want to open it from another computer, just copy this folder and you are ready to go.

Copy the media files you want to use with your project to the according locations, Yonder will automatically add them to your project library when the at the next start.

Generation

Hotkeys

All your hotkeys are displayed here. Activate a hotkey by clicking on it or pressing the key-combination displayed on the button. When pressing down a hotkey, every action defined in Editor will be started and all special actions will be launched. After releasing the hotkey, all actions will be stopped and special actions will be launched again.

When hovering a hotkey, all associated buttons will be highlighted.

Music

In this section you can choose from which playlist music will be played. When a track is playing, its meta information and progress are displayed. When a new playlist is chosen, the track will not change immediatley but the next track will be played from the new playlist. You can force a new track by pressing the »next« button.

Atmosphere

For each defined atmosphere action, one button is being displayed. After activating an atmosphere action, all the tracks of this action will be looped at a random order. Each action always plays only one sound at a time, but you can start as many atmosphere actions as you want simultaniously.

This sound type is great for simulating long and looping sounds like wind, rain or ocean waves.

Singleshots

When a singleshot action is activated, one of the tracks assigned to this action plays one time. After the track has finished, the singleshot action deactivates itself.

Singleshots are particularly nice for events only occurring once, like big explosions or loud screams.

Sfx

One Sfx-object may contain several tracks which will play at a random time in a range you decide on. After a track from a Sfx-object finishes, it will start playing again in the random range.

The Sfx event type ist the most flexible amongst all others. With Sfx events you can easily simulate a battlefield, or a deep jungle with various different sound. If you put some time in configuring your Sfx events, your atmosphere will become much more realistic and credible.

Editor

Start the Ambience Editor by pressing the Edit button or by hitting the hotkey Ctrl + E.

The editing area is basically the same for all event types and Hotkeys.

On the left pane you can see all the objects existing for this event type. Add or remove objects by clicking on the corresponding buttons above the list. You can always rename objects by doubleclicking on them.

On the right pane you can see the whole library for this event type (in the Hotkeys section, this is a bit different. Have a look at the Hotkeys section for further explanation!). There is a small box above the libraries, where you can search for a specific track.

The middle pane displays all tracks you added to an object.

To add a track to an object, drag it from the library to the middle pane.

If you want to remove tracks from an object, select them and hit remove.

To preview a track, you can always drag it on the preview button on the bottom left and it will start playing instantly. Press the preview button to pause or resume the preview.

Sfx

In the Sfx »tracks in object« section, you've got some more options to fine tune your sounds.

A sound in n object will wait for the number of seconds in the colum »pause« before it actually starts playing. To make it more random, you can provide a lower and an upper boundary, in which the pause will be altered.

Example: A sound has a pause of 10 seconds before it will start playing. You povide a lower boundary of 2 seconds and an upper boundary of 5 seconds. The actual pause of the sound will be somewhere between 8 and 15 seconds.

Admittedly, these options are somewhere basic for now, but will improve in future versions of Yonder.

Hotkeys

Admittedly, these options are somewhere basic for now, but will improve in future versions of Yonder.

Drag the objects and special actions you want to use with your hotkey to the middle pane.

When activating a hotkey, its contents will be processed in the order they appear, that means if you want to start playing music off a specific playlist, you will have to change the playlist first, and then start playing music. Otherwise Yonder will start playing from the previous playlist and change to the new one after that.

You may assign a shortcut of your choice to each hotkey. Type the shortcut in the second cloumn of your hotkey objects.

Provide shortcuts in the following format: modifier key + character. You may use as many modifiers and characters as you want, as long as they are concatenated with a + character. Capitalization does not matter.

Note: If a shortcut is already taken by Yonder itself (like Ctrl + Q), it will be omitted.

Modifier keys are:

  • Ctrl
  • Alt
  • Shift
  • Meta (also known as Windows/Super key)

Example: Shortcuts could be: Ctrl+S, Alt+T+I+K, Ctrl+Meta+Alt+N

Resource Browser

The Resource Browser is a small database that helps you finding new music and sfx sounds for your projects.

Start the Resource Browser by pressing the Resource button in the menu bar or by hitting the hotkey Ctrl + R.

Its filtering options make it easy to find the right files for your project. You can enter a searchphrase if you've got an idea of what you are looking for, and filter by the type of sound you need.

If you know of any other good resource, please tell us about it on our forums at yonderfx.com!

Webapp

Yonder comes with a browser based remote control for Ambience, Singleshot and Sfx objects. It can be activated in the configure tab. The webapp is then startet when you are in the generation mode. Press the »webapp running« button to open it in your browser.

To connect to the webapp from another device, type the url displayed in the »webapp running« tooltip. To connect, all devices need to be connected the same local network.

Tricks and tips

  • You don't need to copy all your soundfiles to your project folder. Linking works, too! Unfortunately this makes your project unportable to other machines.
  • Tagging your soundfiles properly makes it much easier to find them by filtering. Nice tag editors are Mp3tag for Windows and EasyTag for Linux.
